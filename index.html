HOW IT WORKS
Real server-side processing with FFmpeg — the industry standard video tool
01
Upload via HTTP
Your video is sent via multipart POST to an Express.js backend. Multer handles the upload and stores it temporarily on the server.
02
FFmpeg Converts
The backend spawns an FFmpeg process with filter_complex to handle blur/mirror/stretch fills and re-encodes as H.264 MP4 in 9:16 portrait.
03
Download Ready
Progress is polled via /api/status. When done, the MP4 is streamed back to you and temp files are cleaned up automatically.
